# fix-mathpix — Clean Up Mathpix OCR LaTeX Output

Mathpix converts scans/PDFs to LaTeX but produces predictable artifacts that must be
fixed before the file is usable. This skill applies all known fixes systematically.

---

## Workflow

1. **Read** the input `.tex` file (from `/mnt/user-data/uploads/` or pasted in chat).
2. **Apply all fixes** listed below, in order.
3. **Write** the cleaned file to `/home/claude/<original-name>-fixed.tex`.
4. **Copy** to `/mnt/user-data/outputs/<original-name>-fixed.tex`.
5. **Present** the file to the user, and briefly summarize what was changed.

---

## Fix Catalogue

Apply **every applicable fix** in a single pass. Do not skip a fix because it seems
minor — each one is there because Mathpix reliably produces that artifact.

---

### FIX 1 — Enumerations (most common artifact)

Mathpix outputs numbered/lettered lists as plain text, not LaTeX environments.
You will see several variants:

**Variant A — multi-line with digits and letters:**
```
1. Text
2. Text
a) Text
b) Text
3. Text
```

**Variant B — letters with dots:**
```
1. Text
a. Text
b. Text
2. Text
```

**Variant C — one-liner with semicolons:**
```
1) a) text; b) text; c) text; 2) text; 3) a) text; b) text
```

**All variants → convert to nested `\begin{enumerate}` with no label arguments:**

```latex
\begin{enumerate}
\item Text
\item Text
\begin{enumerate}
\item Text
\item Text
\end{enumerate}
\item Text
\end{enumerate}
```

Rules:
- Top-level items: `1.`, `1)`, `2.`, `2)` … → `\item` inside outer `enumerate`
- Sub-items: `a)`, `a.`, `b)`, `b.` … → `\item` inside inner `enumerate`
- **Do NOT add `\label` or `[label=...]`** — plain nested enumerate is correct
- For one-liner Variant C, split on `;` first to reconstruct items, then nest
- Preserve all math (`$...$`, `$$...$$`) verbatim inside `\item`

---

### FIX 2 — Itemize (bullet lists)

Mathpix outputs bullet lists as plain dashes or Unicode bullets:

```
- Text lorem ipsum
- Text lorem ipsum
```

or

```
• Text lorem ipsum
• Text lorem ipsum
```

→ Replace with:

```latex
\begin{itemize}
\item Text lorem ipsum
\item Text lorem ipsum
\end{itemize}
```

---

### FIX 3 — Figure environments

Mathpix outputs bare `\includegraphics`:

```latex
\includegraphics[width=0.8\textwidth]{filename.jpg}
```

→ Wrap in a `figure` environment with `[H]` placement and `\centering`:

```latex
\begin{figure}[H]
    \centering
    \includegraphics[scale=0.3]{filename.jpg}
\end{figure}
```

Notes:
- Change `width=...` to `scale=0.3` (standard safe default)
- Keep the filename exactly as-is
- Add `\usepackage{float}` to the preamble if not already present (needed for `[H]`)
- If there is a caption in the surrounding text that clearly belongs to the figure,
  add `\caption{...}` inside the environment

---

### FIX 4 — Stray page numbers

Isolated single numbers on their own line are page numbers leaked from the scan:

```
text or equation
141
text continues
```

→ Remove the isolated number line entirely. A stray page number is identified by:
- A line containing **only** a number (possibly with surrounding whitespace)
- It is NOT inside a math environment (`$`, `$$`, `\[`, `equation`, `align`, etc.)
- It is NOT part of a list item or `\item`
- It is NOT a section/subsection number like `1.0.1`

---

### FIX 5 — Section/chapter headings

Mathpix often outputs chapter and section titles as plain text on their own line
without any LaTeX command:

```
Capitolul 1
Subiecte - sesiunea iulie 2004
1.0.1 Facultatea de Comerţ
```

→ Convert to appropriate LaTeX sectioning commands:

```latex
\chapter{Capitolul 1}
\section{Subiecte - sesiunea iulie 2004}
\subsection{Facultatea de Comerţ}
```

Heuristics for level:
- "Capitolul N", "Chapter N", "Part N" → `\chapter{}`
- "Sesiunea ...", standalone date-like headings → `\section{}`
- Numbered like `1.0.1`, `2.3` → use depth to pick `\section`, `\subsection`, `\subsubsection`
- Faculty/department names that act as sub-headings → `\subsection{}`
- If the document class is `article` (no chapters), use `\section` instead of `\chapter`

---

### FIX 6 — Displayed math spacing

Mathpix sometimes outputs display math with no blank line before/after, causing
it to render inline or with bad spacing:

```latex
text text
$$
formula
$$
text text
```

→ Ensure a blank line before and after `$$...$$` or `\[...\]` blocks:

```latex
text text

$$
formula
$$

text text
```

---

### FIX 7 — Preamble hygiene

Check the preamble and apply these fixes:

| Issue | Fix |
|---|---|
| Missing `\usepackage{float}` but `[H]` used (from Fix 3) | Add it |
| Missing `\usepackage{amsmath}` but `align`/`equation` used | Add it |
| Missing `\usepackage{amssymb}` but `\mathbb`, `\mathcal` used | Add it |
| Missing `\usepackage{graphicx}` but `\includegraphics` used | Add it |
| `\usepackage{inputenc}` with `utf8` — safe to keep | Keep |
| Duplicate `\usepackage{...}` lines | Remove duplicates |
| No `\documentclass` at all | Add `\documentclass{article}` at top |

Do NOT add packages that are not needed.

---

### FIX 8 — Answer-option formatting

Romanian/Romanian-style math exams use lettered answer options. Mathpix often
outputs them without proper structure. If options appear as:

```
а) $\alpha = -\frac{3}{25}$;
b) $\alpha = \frac{17}{25}$;
```

(mixing Cyrillic "а" with Latin "b", "c", etc. — this is a known Mathpix OCR quirk)

→ Normalize all option labels to Latin letters and wrap as a nested enumerate:

```latex
\begin{enumerate}
\item $\alpha = -\frac{3}{25}$
\item $\alpha = \frac{17}{25}$
\end{enumerate}
```

Remove trailing semicolons from the last item.

---

### FIX 9 — Stray backtick / code fence artifacts

If Mathpix was run through a markdown-exporting pipeline, you may see:

````
```latex
...content...
```
````

→ Strip the fences; keep only the LaTeX content inside.

---

### FIX 10 — Unicode characters in text

Mathpix correctly OCRs diacritics (ţ, ş, ă, â, î, etc.) and outputs them as
UTF-8. This is fine — keep them as-is if `inputenc` with `utf8` is in the
preamble. If there is **no** `inputenc` package, add:

```latex
\usepackage[utf8]{inputenc}
```

Do NOT escape diacritics to `\c{t}`, `\c{s}` etc. unless the user explicitly asks.

---

## What NOT to change

- Math content inside `$...$`, `$$...$$`, `\[...\]`, `equation`, `align` — never alter
- Existing correct LaTeX environments already in the file
- Text content, wording, or language
- Filenames in `\includegraphics`
- The logical structure/ordering of problems
